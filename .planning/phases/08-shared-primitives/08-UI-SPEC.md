# Phase 8 — UI Design Contract (Shared Primitives Restyle)

> Visual and interaction contract for the retrofit restyle of 8 shadcn primitives. No new components, no new routes, no prop contract changes. Generated by gsd-ui-researcher from CONTEXT D-01..D-23, RESEARCH §Recipes and §9.1 remediation, and DESIGN.md.

**Scope:** `packages/ui/src/shadcn/{button,card,input,textarea,select,badge,avatar,sheet}.tsx` only. Token VALUES are frozen (Phase 7 owns them); Phase 8 swaps CLASSES (shape, padding, radius, gradient, focus recipe) only.

## Spacing Scale

Tailwind 4 default (4px base unit). Phase 8 primitives draw from this canonical set; no custom spacing is added this phase.

| Token | Value | Usage in Phase 8 primitives |
|-------|-------|------------------------------|
| 0.5  | 2px   | (unused) |
| 1    | 4px   | Base icon gap |
| 1.5  | 6px   | Button `sm` vertical padding (`py-1.5`) |
| 2    | 8px   | `gap-2` between icon + label in Button base, Badge horizontal rhythm |
| 3    | 12px  | **Form primitive vertical padding (`py-3`) — PRIM-03 anchor** |
| 4    | 16px  | Sheet header/footer gap + padding (`pb-4`, `pt-4`), Input horizontal padding (`px-4`), Button `default` horizontal padding seed |
| 5    | 20px  | Button `default` horizontal padding (`px-5`) |
| 6    | 24px  | Button `lg` horizontal padding (`px-6`), Sheet content padding (`p-6` default) |
| 8    | 32px  | Button `pill` horizontal padding (`px-8`) |

**Radius scale (Phase 7 `--radius: 1rem`):**

| Token | Rem / Px | Usage |
|-------|----------|-------|
| `rounded-xl`  | 0.75rem / 12px | Button `sm` size |
| `rounded-2xl` | 1rem / 16px    | **Hero radius** — Button default/lg/icon, Card, Input, Textarea, Select, Sheet leading corner |
| `rounded-full`| 9999px         | Badge pills, Avatar, Button `pill` variant |

**Touch target floor:** `py-3 text-base` on form primitives produces intrinsic height ≈ 48px (16px line-height × 1.5 + 12px top + 12px bottom = 48px), exceeding the WCAG 2.5.5 44×44px minimum. Button `default` at `px-5 py-3 text-sm` produces ≈ 44px. Button `sm` at `py-1.5 text-xs` intentionally falls below 44px and is reserved for in-row / secondary contexts only (no new `sm`-size callers added this phase).

**Exceptions:** None beyond Phase 7 defaults.

## Typography

Inter Variable, loaded globally via Phase 7. Phase 8 touches the following roles:

| Role | Size | Weight | Line Height | Used by |
|------|------|--------|-------------|---------|
| Form body (primitives) | 16px (`text-base`) | 400 | 1.5 | Input, Textarea, SelectTrigger |
| Button label (default/lg) | 14px (`text-sm`) | 500 (`font-medium`) | 1.25 | Button base |
| Button label (sm) | 12px (`text-xs`) | 500 | 1.0 | Button `sm` |
| Badge label | 12px (`text-xs`) | 500 (`font-medium`) | 1.0 | All Badge variants |
| Sheet title | 18px (`text-lg`) | 600 (`font-semibold`) | 1.2 | `SheetTitle` (discretion: bump from current `font-medium`) |
| Card title | 14px (`text-sm`) inherited | 400 (unchanged) | 1.25 | `CardTitle` (not touched — RESEARCH §Open Q3) |
| Avatar fallback initials | 12/14/16px per `size` (`text-xs` / `text-sm` / `text-base`) | 500 (`font-medium`) | 1.0 | `AvatarFallback` |

**Rules:**
- Form primitives step UP from `text-sm` (14px) → `text-base` (16px) to satisfy PRIM-03 and prevent iOS zoom-on-focus.
- Button labels stay at `text-sm` / `text-xs` — padding carries the visual heft, not type size.
- No fluid typography; sizes are fixed per role.

## Color

Phase 8 primitives are **100% token-driven** with exactly one deliberate exception: the brand gradient. No `slate-*`, `gray-*`, `white`, or `black` literals are permitted in class strings **except** the gradient and its companion `text-white`.

### Token roles (values owned by Phase 7 — read-only here)

| Role | Light | Dark | Used by |
|------|-------|------|---------|
| Dominant (60%) — `--background` | `#f1f5f9` | `#0f172a` | Page canvas, Button `outline`/`secondary` fill, Input fill |
| Secondary (30%) — `--card` | `#ffffff` | `#1e293b` | Card surface, **Sheet surface (changed from `background`)** |
| Accent (10%) — `--primary` / `--ring` | `#22c55e` | `#4ade80` | Focus ring, Badge `default` text accent (token), Button `brand` retune gradient anchor |
| Muted surface — `--muted` | `#f1f5f9` | `#1e293b` | Badge `default`/`secondary` fill, Button secondary hover |
| Muted text — `--muted-foreground` | `#64748b` | `#94a3b8` | Placeholder, Badge secondary text |
| Border — `--border` | `#e2e8f0` | `#334155` | **ALL** primitive borders (replaces `border-input` where used) |
| Semantic red bg/fg | `#fef2f2` / `#dc2626` | `rgba(239,68,68,.15)` / `#f87171` | Badge `destructive` |
| Semantic amber bg/fg | `#fffbeb` / `#d97706` | `rgba(245,158,11,.15)` / `#fbbf24` | Badge `warning` |
| Semantic green bg/fg | `#f0fdf4` / `#16a34a` | `rgba(34,197,94,.15)` / `#4ade80` | Badge `success` |
| Semantic blue bg/fg | `#eff6ff` / `#2563eb` | `rgba(59,130,246,.15)` / `#60a5fa` | Badge `info` |

### Brand gradient (literal — the ONE exception)

| Surface | Class string |
|---------|--------------|
| Button `default` / `brand` / `pill` face | `bg-gradient-to-r from-green-500 to-emerald-600 text-white shadow-lg shadow-green-500/25 hover:shadow-xl` |
| Avatar fallback | `bg-gradient-to-br from-green-500 to-emerald-600 text-white font-medium` |

**Note:** Avatar uses `to-br` (135°); Button uses `to-r` (0°). This matches the prototype. Both keep the gradient IDENTICAL across light and dark themes by design (D-06, D-19).

### Accent reserved for (explicit list)

The 10% accent (`--primary` / green-500 family) is restricted to:
1. Focus ring on all interactive primitives (`ring-primary`).
2. Primary button face (via gradient — literal brand color).
3. Avatar initials fallback (via gradient).
4. Badge `default` variant is NOT an accent surface in Phase 8 — retuned to `bg-muted text-muted-foreground` per R2 to remediate §9.1 #9 (2.28:1 contrast failure).

**Nothing else.** No links, icons, or status text in these 8 primitives use accent green directly.

### Destructive

| Use | Treatment |
|-----|-----------|
| Button `destructive` variant | Unchanged — `bg-destructive text-destructive-foreground hover:bg-destructive/90` |
| Badge `destructive` variant | **Retuned** to semantic pastel: `bg-semantic-red-bg text-semantic-red-fg` (consistency with success/warning/info) |
