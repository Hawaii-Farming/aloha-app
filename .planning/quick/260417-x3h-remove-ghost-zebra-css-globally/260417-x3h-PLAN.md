# Quick Task 260417-x3h

## Problem

Every AG Grid list view paints striped bands in any empty area — below the last data row (short datasets, "No records found" states) and to the right of the last column when columns don't fill the grid width. Root cause is a `repeating-linear-gradient` painted on `.ag-body-viewport` in `app/styles/kit.css` ("ghost zebra fill"). It was added so short datasets wouldn't show a flat dead zone, but it actually looks worse than a flat empty area (especially in Employee Review with zero rows and Scheduler with one row).

Per-grid workarounds (domLayout=autoHeight, sizeColumnsToFit) fix one view at a time and don't scale — the list views NEED the grid to fill height for scrolling behavior, they just don't want the stripes below the data.

## Fix

1. `app/styles/kit.css` — delete the `[data-ag-theme-mode='light'].ag-grid-cell-borders .ag-body-viewport { background-image: repeating-linear-gradient(...) }` rule.
2. `UI-RULES.md` — remove the `**Ghost zebra fill**` bullet from §Tables; convention no longer applies.

Real row zebra is unchanged (provided by AG Grid theme `oddRowBackgroundColor` in `ag-grid-theme.ts`). Empty area now renders as flat grid background like Employee Review's "No records found" state should have looked all along.

## Verification

`pnpm typecheck` clean. All list views: data rows keep zebra, empty area is flat.
