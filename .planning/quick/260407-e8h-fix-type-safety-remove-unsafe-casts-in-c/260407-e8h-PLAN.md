**Fix Category 3 (fetcher.submit data casts, 3 occurrences):**

React Router's `fetcher.submit()` first argument is `SubmitTarget` which accepts `JsonValue`. `JsonValue = JsonPrimitive | JsonObject | JsonArray` where `JsonObject = { [Key in string]: JsonValue }`. The problem is `Record<string, unknown>` doesn't satisfy `JsonObject` because `unknown` is not assignable to `JsonValue`.

The fix: Change the `onSubmit` callback parameter type from `Record<string, unknown>` to `Record<string, string | number | boolean | null>` (which satisfies `JsonObject`). This is safe because form data values are always JSON-serializable primitives.

In `app/routes/workspace/sub-module-create.tsx` line 193:
- Change `onSubmit` parameter from `(data: Record<string, unknown>)` to `(data: Record<string, string | number | boolean | null>)`.
- Remove the `data as unknown as Record<string, string>` cast -- pass `data` directly to `fetcher.submit()`.

In `app/components/crud/card-detail-view.tsx` line 216:
- The `payload` object has nested `transitionFields` which may contain objects. The fix here: type `payload` explicitly as a `Record<string, string | number | boolean | null | Record<string, string | number | boolean | null>>` or use a JSON-compatible type. Alternatively, since `encType: 'application/json'` serializes to JSON, wrap with `JSON.stringify` and use `encType: 'application/json'` with body as string -- but that changes the API contract.
- Simpler approach: Create a local type alias `type JsonRecord = Record<string, string | number | boolean | null | Record<string, string | number | boolean | null>>` and assert `payload satisfies JsonRecord` to get compile-time checking, then pass to submit. If `transitionFields` is too deeply nested, the simplest safe fix is to use `JSON.parse(JSON.stringify(payload))` as the submit target (this creates a value TypeScript knows is JSON-safe).
- Preferred: Import `JsonValue` from react-router (it is exported) and use that: `fetcher.submit(payload as JsonValue, ...)`. This is a single-hop cast that is semantically correct (we know the payload is JSON-serializable). This is much better than `as unknown as Record<string, string>`.

In `app/components/crud/create-panel.tsx` line 96:
- Same pattern as sub-module-create.tsx. Fix the callback parameter type and remove the cast.
