# Quick Task 260418-tz3

## Objective

Replace the native `<input type="time">` in `FormDateTimeField` with two Shadcn `<Select>` dropdowns (HH + MM) so the time picker matches the look and feel of every other dropdown in the form.

Native time-input on macOS renders a stacked scrolling spinner (screenshot shared) — ugly and inconsistent.

## Design

`[ Calendar button ......... ] [ HH ▾ ] : [ MM ▾ ]`

- **Hour:** 00–23, 24 options (24-hour clock, matches existing `HH:mm` format).
- **Minute:** 15-min increments — `00 15 30 45` — 4 options. Shift schedules almost always land on the quarter hour; finer granularity can be added later.
- Both `<Select>` use the standard `SelectTrigger` styling already used elsewhere.
- `w-20` or tighter on each trigger to keep the row compact.

## Changes

### `packages/ui/src/kit/form-fields.tsx`

- Remove `<Input type="time">`.
- Remove unused `Input` import from this file **only if no other consumer** (grep first).
- Build HH (24) + MM (4) option arrays as module-level constants.
- Each Select's `onValueChange` combines with the other dimension + the current calendar date, calls `field.onChange(formatISO(...))`.
- Keep the combined storage format unchanged (`formatISO` output).

## Verification

- `pnpm typecheck` clean.
- Scheduler form manual check: clicking HH or MM shows standard Shadcn dropdown, selection updates the stored datetime, reopen shows the same values.
