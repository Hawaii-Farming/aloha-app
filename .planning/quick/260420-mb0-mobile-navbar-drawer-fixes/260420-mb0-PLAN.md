<objective>
Fix four mobile UX regressions in the workspace shell:

1. Restore a search affordance on the mobile header (currently absent <md).
2. Fix the mobile avatar so it shows the current org's initials (not a brand "A").
3. Make the mobile avatar interactive — open the same profile dropdown used on desktop.
4. Smooth the create-panel Sheet open animation on mobile by removing
   `backdrop-blur-sm` from the Sheet/Dialog/AlertDialog overlays. Keep the
   `bg-black/60` scrim so there is still a clear backdrop. Rationale:
   `backdrop-blur-sm` forces the compositor to run a blur shader on every
   animation frame — on iOS Safari / lower-end Android this thrashes the GPU
   and makes the slide-in feel janky. `bg-black/60` alone composites cheaply.

Purpose: Ship mobile parity for the navbar and fix a perf papercut before
the next design-branch merge. No feature work, no new deps.

Output: A mobile header that matches desktop navbar functionality (search +
profile menu with correct org initials), and a smoother overlay animation
shared by Sheet/Dialog/AlertDialog on mobile.
</objective>

  In each of the three files, find the overlay className that contains
  `bg-black/60 backdrop-blur-sm` and delete the ` backdrop-blur-sm` token
  (leave `bg-black/60` and every other class unchanged, including the
  `data-[state=...]:animate-in/out`, `fade-in-0`, `fade-out-0`, and `fixed
  inset-0 z-50` tokens).

  Exact current string in all three files (the executor should match
  verbatim to avoid collateral edits):

  ```
  'data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 fixed inset-0 z-50 bg-black/60 backdrop-blur-sm'
  ```

  Change to:

  ```
  'data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 fixed inset-0 z-50 bg-black/60'
  ```

  Rationale (do not add as a code comment — keep it in the commit message):
  `backdrop-blur-sm` forces a full-viewport blur composition pass every
  animation frame. On mobile Safari and mid-range Android this blocks the
  compositor thread during the Sheet slide-in, producing a visible stutter
  before the sheet settles. `bg-black/60` is a flat alpha layer —
  GPU-cheap, indistinguishable as a backdrop at 60% opacity, and used the
  same way by default shadcn-ui overlays today.

  Do NOT change:
  - Dark-mode variants or token values
  - Animation durations, easings, or keyframes
  - z-index, position, or sizing tokens
  - Anything in `SheetContent` / `DialogContent` / `AlertDialogContent`
  - Any other file under `packages/ui/src/shadcn/`

  Consistency matters: the user's create-panel is a Sheet, but confirm
  dialogs (delete row) use AlertDialog and other flows use Dialog. Keeping
  all three overlays identical avoids future "why does only one feel
  janky?" regressions. If pnpm lint flags any ordering change (Prettier
  plugin reorders Tailwind classes), accept the reorder — it is cosmetic
  and does not affect the removal.
  </action>
