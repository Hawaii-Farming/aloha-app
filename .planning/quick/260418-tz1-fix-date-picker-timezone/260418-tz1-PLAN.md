# Quick Task 260418-tz1

## Objective

Fix timezone off-by-one bug in `FormDateField`. Clicking 4/16 in the calendar displays and re-selects 4/15 in HST (UTC-10).

## Root cause

`packages/ui/src/kit/form-fields.tsx` lines 205 and 213 parse the stored `'yyyy-MM-dd'` string with `new Date(field.value)`. JavaScript parses date-only ISO strings as **UTC midnight**. In HST that renders as the prior day 14:00.

## Fix

Replace `new Date(field.value)` with `parse(field.value, 'yyyy-MM-dd', new Date())` from `date-fns` — parses as local time, preserving the selected calendar day.

- Line 205 (display): `format(new Date(field.value), 'MM/dd/yyyy')` → `format(parse(field.value, 'yyyy-MM-dd', new Date()), 'MM/dd/yyyy')`
- Line 213 (selected): `new Date(field.value)` → `parse(field.value, 'yyyy-MM-dd', new Date())`
- Add `parse` to existing `date-fns` import.
