Create two new CRUD config files following the exact pattern from invnt-item.config.ts.

**org-site.config.ts** (warehouses submodule):
- tableName: 'org_site', pkType: 'text', pkColumn: 'id', orgScoped: true
- views: { list: 'org_site', detail: 'org_site' }
- Zod schema fields: id (string, required), name (string, required), category (string, required), subcategory (string, optional), acres (number, optional), total_rows (number, optional), avg_units_per_row (number, optional), notes (string, optional), is_food_contact_surface (boolean, default false), zone (string, optional)
- columns: id (sortable), name (sortable), category (sortable), zone, acres (number), is_food_contact_surface (boolean), created_at (datetime, sortable)
- search: columns ['name', 'category', 'zone'], placeholder 'Search warehouses...'
- filters: category (select, options: determine from schema — use type select without hardcoded options so it auto-populates), zone (select, options: ['zone_1', 'zone_2', 'zone_3', 'zone_4'])
- formFields: id (text, required, showOnCreate only), name (text, required), category (text, required), subcategory (text), acres (number), total_rows (number), avg_units_per_row (number), zone (select, options: ['zone_1', 'zone_2', 'zone_3', 'zone_4']), is_food_contact_surface (boolean), notes (textarea)

**invnt-onhand.config.ts** (stock_counts submodule):
- tableName: 'invnt_onhand', pkType: 'uuid', pkColumn: 'id', orgScoped: true
- views: { list: 'invnt_onhand', detail: 'invnt_onhand' }
- Zod schema fields: invnt_item_id (string, required), onhand_date (string/date, required), onhand_uom (string, optional), onhand_quantity (number, required), burn_per_onhand (number, default 0), invnt_lot_id (string, optional), notes (string, optional)
- Note: id is UUID auto-generated, so do NOT include id in schema or as a create form field
- columns: invnt_item_id (sortable), onhand_date (date, sortable), onhand_quantity (number, sortable), onhand_uom, burn_per_onhand (number), created_at (datetime, sortable)
- search: columns ['invnt_item_id'], placeholder 'Search stock counts...'
- filters: none initially
- formFields: invnt_item_id (fk, fkTable: 'invnt_item', fkLabelColumn: 'name', required), onhand_date (date, required), onhand_quantity (number, required), onhand_uom (fk, fkTable: 'sys_uom', fkLabelColumn: 'id'), burn_per_onhand (number), invnt_lot_id (fk, fkTable: 'invnt_lot', fkLabelColumn: 'id'), notes (textarea)

Both files must use the same import style and export pattern as invnt-item.config.ts.
