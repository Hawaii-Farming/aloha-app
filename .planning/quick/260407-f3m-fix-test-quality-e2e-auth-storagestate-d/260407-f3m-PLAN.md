```typescript
function createMockSupabaseChain() {
  const calls: { method: string; args: unknown[] }[] = [];

  // Each method records its call and returns the chain (or a terminal value for 'range')
  const chain: Record<string, (...args: unknown[]) => unknown> = {};

  const methods = ['select', 'eq', 'is', 'not', 'or', 'ilike', 'order', 'in'] as const;

  for (const method of methods) {
    chain[method] = (...args: unknown[]) => {
      calls.push({ method, args });
      return chain;
    };
  }

  chain.range = (...args: unknown[]) => {
    calls.push({ method: 'range', args });
    return Promise.resolve({ data: [], count: 0, error: null });
  };

  const client = {
    from: (table: string) => {
      calls.push({ method: 'from', args: [table] });
      return chain;
    },
  };

  return { client: client as unknown as SupabaseClient<Database>, calls };
}
```

Import `SupabaseClient` from `@supabase/supabase-js` and `Database` from `~/lib/database.types` at the top of the test file.

Update all test usages:
- Replace `createMockClient()` calls with `createMockSupabaseChain()`
- Remove the `client as never` cast from all `loadTableData()` calls — the factory now returns a properly typed client
- Keep the `await getLoadTableData()` dynamic import pattern (needed for .server.ts module)

The key improvement: explicit method list instead of Proxy catch-all, and a proper type assertion at the factory boundary instead of at every call site.

Wait — the test file header already says "auth_link_log not readable by authenticated role" in the comment (line 7) but there is no test for it. The plan count is 10 but there are only 10 SELECT assert calls. We need to bump to 11 and add the RLS test.

The standard pgTAP pattern for testing RLS is to switch role using `SET LOCAL ROLE authenticated` and set `request.jwt.claims` to impersonate a user, then verify the row count is 0.

However, since `auth_link_log` likely has NO `SELECT` grant to `authenticated` at all (no RLS policy needed — just no grant), the simplest test is:

```sql
-- Switch to authenticated role to test access
SET LOCAL ROLE authenticated;

-- Set JWT claims to a known test user
SET LOCAL request.jwt.claims = '{"sub": "11111111-1111-1111-1111-111111111002", "role": "authenticated"}';

SELECT is(
  (SELECT count(*)::integer FROM public.auth_link_log),
  0,
  'RLS: authenticated role cannot read auth_link_log'
);

-- Reset to superuser for finish()
RESET ROLE;
```

If the table has no SELECT grant, this will return 0 rows (permission denied is converted to empty result set under RLS). If it throws an error instead, wrap in a `lives_ok` or use `throws_ok` with the permission denied message. The most likely behavior with RLS enabled and no policy is 0 rows returned.

Place this block between Test 6 and `SELECT * FROM finish()`. Update plan count to 11.
