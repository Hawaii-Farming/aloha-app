10. --sidebar-foreground oklch(35% 0 none) on --sidebar-background oklch(99% 0 none)

12. --sidebar-accent-foreground oklch(30% 0 none) on --sidebar-accent oklch(95% 0.01 160)

**Pairs to verify (3:1 minimum for UI components):**
13. --ring oklch(47% 0.165 160) against --background oklch(99% 0 none)
14. --border oklch(89% 0 none) against --background oklch(99% 0 none)
15. --input oklch(85% 0 none) against --background oklch(99% 0 none)

To compute contrast ratios from oklch, convert to relative luminance:
- oklch L% roughly maps to perceptual lightness. For WCAG sRGB contrast:
  - oklch(99%) ~= sRGB luminance 0.95
  - oklch(96%) ~= sRGB luminance 0.90
  - oklch(89%) ~= sRGB luminance 0.75
  - oklch(85%) ~= sRGB luminance 0.67
  - oklch(45%) ~= sRGB luminance 0.15
  - oklch(35%) ~= sRGB luminance 0.09
  - oklch(30%) ~= sRGB luminance 0.06
  - oklch(12%) ~= sRGB luminance 0.01
  - oklch(47% 0.165 160) ~= sRGB luminance 0.14 (green)

WCAG contrast ratio = (L1 + 0.05) / (L2 + 0.05) where L1 > L2.

Calculate each pair. If ANY pair fails:
- For text pairs failing 4.5:1: adjust the darker color to lower oklch L% or adjust the lighter color to higher oklch L%
- For UI component pairs failing 3:1: adjust similarly
- Document adjustments in a brief table

If all pass, document the results in a contrast verification table.

**Also verify:**
- app/styles/global.css has `:root { --shadow: none; ... }` (shadow nullification)
- app/styles/shadcn-ui.css .dark block is unchanged from Phase 1
