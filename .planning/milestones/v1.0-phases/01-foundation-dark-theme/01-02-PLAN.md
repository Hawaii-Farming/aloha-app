2. Edit `app/styles/global.css` — add shadow removal rules for dark mode. Add the following INSIDE the existing `@layer base` block, after the `textarea::placeholder` rule (per D-10, D-11, D-12):

```css
.dark {
  --shadow: none;
  --shadow-sm: none;
  --shadow-md: none;
  --shadow-lg: none;
  --shadow-xl: none;
  --shadow-2xl: none;
}
```

This overrides Tailwind CSS 4 shadow tokens to `none` in dark mode, removing all box-shadows from cards, containers, alert-dialogs, and sheets without editing any .tsx component files. The `@layer base` scope ensures this runs at the correct CSS cascade level.

Per D-12, focus ring shadows are preserved because focus rings use the `--ring` token and `ring-*` utilities (which use `box-shadow` with `--ring-*` tokens, NOT the `--shadow-*` tokens). The `--ring` token was set to Supabase green in Plan 01.

NOTE: If the Tailwind CSS 4 `--shadow-*` token override approach does not work as expected at runtime (Assumption A3 from RESEARCH.md), the fallback is to add `.dark [class*="shadow"]` CSS rules. Verify with `pnpm dev` that shadows are actually removed.
  </action>
