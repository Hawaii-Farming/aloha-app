| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | denial_reason via extraFields | mitigate | denial_reason is a TEXT column; Supabase SDK uses parameterized queries preventing SQL injection; value length is implicitly limited by database TEXT type |
| T-03-08 | Elevation of Privilege | approve/deny without permission | accept | Module-level access via hr_module_access enforced in app layer; RLS ensures org isolation; any org member with module access can approve/deny (business rule) |
| T-03-09 | Spoofing | requested_by auto-set | mitigate | Server-side only -- employeeId from authenticated session, not client input (D-21) |
| T-03-10 | Tampering | filter_status URL param | accept | loadTableData validates param against allowedColumns; value is used in parameterized .eq() query via Supabase SDK -- no injection risk |
</threat_model>
