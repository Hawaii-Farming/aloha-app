  <action>
Create migration file `supabase/migrations/20260410000001_app_hr_hours_comparison.sql` with a `CREATE OR REPLACE VIEW app_hr_hours_comparison` that:

1. **CTE `schedule_agg`**: Aggregates `ops_task_schedule` by `(org_id, hr_employee_id, pay_period_start, pay_period_end)`:
   - Cross-join with distinct pay periods from `hr_payroll` (WHERE `is_deleted = false`) to bucket schedule entries
   - Join condition: `s.org_id = p.org_id AND s.start_time::DATE >= p.pay_period_start AND s.start_time::DATE <= p.pay_period_end`
   - Hours formula per D-03: `ROUND(SUM(CASE WHEN s.stop_time IS NOT NULL THEN EXTRACT(EPOCH FROM (s.stop_time - s.start_time)) / 3600.0 ELSE 0 END)::NUMERIC, 2) AS scheduled_hours`
   - Filter: `s.is_deleted = false AND s.start_time IS NOT NULL`
   - Include ALL entries regardless of `ops_task_tracker_id` (per D-02)
   - NULL `stop_time` contributes 0 hours (per D-06)

2. **CTE `payroll_agg`**: Aggregates `hr_payroll` by `(org_id, hr_employee_id, pay_period_start, pay_period_end)`:
   - `SUM(total_hours) AS payroll_hours`
   - Filter: `is_deleted = false`

3. **Main SELECT**: FULL OUTER JOIN `schedule_agg` with `payroll_agg` on `(org_id, hr_employee_id, pay_period_start, pay_period_end)`:
   - COALESCE org_id, hr_employee_id, pay_period_start, pay_period_end from both sides
   - JOIN `hr_employee e ON e.id = COALESCE(sa.hr_employee_id, pa.hr_employee_id)` (WHERE `e.is_deleted = false`)
   - LEFT JOIN `hr_department d ON d.id = e.hr_department_id`
   - Select columns per D-04: `org_id`, `hr_employee_id`, `pay_period_start`, `pay_period_end`, `e.first_name || ' ' || e.last_name AS full_name`, `e.profile_photo_url`, `d.name AS department_name`, `COALESCE(sa.scheduled_hours, 0) AS scheduled_hours`, `COALESCE(pa.payroll_hours, 0) AS payroll_hours`, `ROUND((COALESCE(pa.payroll_hours, 0) - COALESCE(sa.scheduled_hours, 0))::NUMERIC, 2) AS variance`

4. **Grant**: `GRANT SELECT ON app_hr_hours_comparison TO authenticated;`

**Create `app/routes/api/schedule-by-period.ts`** following the `schedule-history.ts` pattern:

1. Export `loader` function with signature `async ({ request }: { request: Request })`
2. Extract URL searchParams: `employeeId`, `orgId`, `periodStart`, `periodEnd`
3. Validate: return 400 if any of `orgId`, `employeeId`, `periodStart`, `periodEnd` is missing
4. Query `ops_task_schedule` using `getSupabaseServerClient(request)`:
   ```
   client.from('ops_task_schedule' as never)
     .select('id, start_time, stop_time, hr_employee_id, org_id, hr_employee!inner(hr_department(name)), ops_task!inner(name)')
     .eq('org_id', orgId)
     .eq('hr_employee_id', employeeId)
     .eq('is_deleted', false)
     .not('start_time', 'is', null)
     .gte('start_time', `${periodStart}T00:00:00`)
     .lte('start_time', `${periodEnd}T23:59:59`)
     .order('start_time', { ascending: true })
   ```
5. Map rows to enriched format (same pattern as schedule-history.ts detail mode):
   - `date`: `startDate.toISOString().split('T')[0]`
   - `day_of_week`: `startDate.toLocaleDateString('en-US', { weekday: 'short' })`
   - `department_name`: from `hr_employee.hr_department.name`
   - `task_name`: from `ops_task.name`
   - `start_time_formatted`: HH:mm 24h format
   - `end_time_formatted`: HH:mm 24h format
   - `hours`: `Math.round(((stopDate - startDate) / 3600000) * 100) / 100` (null if stop_time is null)
6. Return `Response.json({ data: enriched })`
7. On query error: `Response.json({ error: error.message }, { status: 500 })`

**Update `app/routes.ts`**: Add route registration line after the `schedule-history` line:
